# Sketch RTS Spec Coverage

This file is the checkpoint ledger for proof, not a second spec. The spec remains `2026-05-30-sketch-rts-design.md`.

## What Counts

Browser E2E means the real app in a real browser, driven by mouse, keyboard, DOM clicks, and canvas sampling. It may use REST only to reset or seed a scenario when the point under test is the browser-facing command/rendering path. It cannot use SDK/API calls as the action being claimed for player controls.

SDK/API tests are agent-surface proof. Simulation tests are rules proof. Both matter, but neither replaces Browser E2E.

## Current Weak Line

Some browser proofs seed small scenarios through REST so the test can reach rare states quickly. That is allowed only as setup. The action under player-control test must still happen through mouse, keyboard, or DOM command buttons, and the visible result must be read from the real page/canvas.
